# SpecKit Quick Start Guide

## What is SpecKit?

GitHub SpecKit is a specification-driven development toolkit that helps you:
- Write formal specifications that AI agents can understand
- Break down work into structured plans and tasks
- Track implementation progress
- Maintain consistency across conversions

## Installation (5 minutes)

### Step 1: Install SpecKit CLI

```bash
# Option 1: Using uv (recommended)
uv tool install spec-kit

# Option 2: Using pip
pip install spec-kit

# Option 3: Using npm
npm install -g spec-kit
```

### Step 2: Initialize in Your Project

```bash
cd C:\source\agents\ClaudeOnshoreConversionAgent
specify init . --ai claude --script ps
```

This creates:
- `.specify/` - Configuration directory
- `specs/` - Where your specs will live

### Step 3: Create Constitution (Optional but Recommended)

Edit `.specify/constitution.md`:

```markdown
# ClaudeOnshoreConversionAgent Constitution

## Principles
1. Preserve all business logic from VB.NET
2. Follow BargeOps.Admin.Mono patterns
3. Create Shared DTOs first
4. One entity at a time, fully tested
```

## Quick Example: Vendor Entity

### 1. Run Analysis (Existing)
```bash
bun run agents/orchestrator.ts --entity "Vendor" --form-name "frmVendorSearch"
```

### 2. Create SpecKit Spec (New)

**Option A: Manual** - Create `specs/Vendor/spec.md` using the example in `docs/SPECKIT_EXAMPLE_SPEC.md`

**Option B: Automated** - (Future) Run spec generator:
```bash
bun run agents/spec-generator.ts --entity "Vendor"
```

### 3. Create Plan (In Claude Code)
```
/speckit.plan
```

This reads `specs/Vendor/spec.md` and creates `specs/Vendor/plan.md` with:
- Technical stack decisions
- Architecture patterns
- API design
- UI component choices

### 4. Break Down Tasks (In Claude Code)
```
/speckit.tasks
```

This creates `specs/Vendor/tasks.md` with:
- Task 1: Create Shared DTOs
- Task 2: Implement Repository
- Task 3: Create API Service
- Task 4: Build API Controller
- Task 5: Create ViewModels
- Task 6: Build MVC Controller
- Task 7: Create Razor Views
- Task 8: Add JavaScript
- Task 9: Testing
- Task 10: Deployment

### 5. Generate Templates (Existing)
```bash
bun run generate-template-api --entity "Vendor"
bun run generate-template-ui --entity "Vendor"
```

Templates can now reference SpecKit specs for better context.

## SpecKit Commands Cheat Sheet

### In Claude Code

| Command | What It Does | When to Use |
|---------|--------------|-------------|
| `/speckit.constitution` | Define project principles | Once at setup |
| `/speckit.specify` | Create functional spec | After analysis |
| `/speckit.clarify` | Clarify requirements | If spec unclear |
| `/speckit.plan` | Create technical plan | After spec |
| `/speckit.tasks` | Break down tasks | After plan |
| `/speckit.implement` | Execute implementation | After tasks |
| `/speckit.analyze` | Analyze code quality | During/after implementation |
| `/speckit.checklist` | Validation checklist | Before completion |

### CLI Commands

```bash
# Initialize project
specify init . --ai claude

# Check if SpecKit is set up correctly
specify check

# Get help
specify --help
```

## Integration Strategies

### Strategy 1: Minimal (Recommended to Start)
- Keep existing workflow
- Add SpecKit specs manually for important entities
- Use SpecKit for planning and task breakdown
- Templates still generated by existing agents

**Pros:** Low risk, easy to adopt
**Cons:** Manual spec creation

### Strategy 2: Hybrid
- Create `spec-generator.ts` agent
- Auto-generate specs from analysis JSON
- Use SpecKit for planning and tasks
- Templates reference SpecKit specs

**Pros:** Automated, consistent
**Cons:** Requires development

### Strategy 3: Full Integration
- Replace conversion plans with SpecKit
- All entities use SpecKit workflow
- Template generators read SpecKit specs

**Pros:** Fully standardized
**Cons:** More work upfront

## Tips

1. **Start Small**: Try SpecKit on one entity first (e.g., Vendor)
2. **Use Examples**: Copy `SPECKIT_EXAMPLE_SPEC.md` as a template
3. **Iterate**: Specs can be updated as you learn more
4. **Combine**: Use SpecKit for planning, keep existing template generators
5. **Document**: Add notes to specs as you implement

## Troubleshooting

### "specify: command not found"
**Fix:** Install SpecKit: `uv tool install spec-kit` or add to PATH

### Commands not working in Claude Code
**Fix:** Ensure `.specify/` directory exists (run `specify init`)

### Specs not aligning with analysis
**Fix:** Manually review and update spec based on JSON files
